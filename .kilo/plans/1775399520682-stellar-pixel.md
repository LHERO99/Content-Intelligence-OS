## Plan: Implement Single-Column Ascending/Descending Sorting

The user wants to restrict table sorting in the Content-Planning tabs to only allow single-column ascending or descending order. This means removing the ability for multi-column sorting and removing the "no sort" state (tri-state sorting). The icons in the URL History popup should all be in the primary color.

### Problem Identification:
Tables in the application use `@tanstack/react-table`. By default, `useReactTable` allows multi-sorting and a tri-state sorting cycle (ascending, descending, none). The current implementation in `src/app/content-history-table.tsx`, `src/app/planning/keyword-table.tsx`, and `src/app/planning/suggestions-table.tsx` does not explicitly configure these options, thus retaining the default behavior.

### Proposed Changes:
For each of the identified table components, I will modify their `useReactTable` configuration to enforce single-column ascending/descending sorting:

1.  **`src/app/content-history-table.tsx`**:
    *   Add `enableMultiSort: false` to disable sorting on multiple columns simultaneously.
    *   Add `enableSortingRemoval: false` to ensure that clicking a sorted column toggles only between ascending and descending (removing the "none" or unsorted state).

2.  **`src/app/planning/keyword-table.tsx`**:
    *   Apply the same `enableMultiSort: false` and `enableSortingRemoval: false` configurations.

3.  **`src/app/planning/suggestions-table.tsx`**:
    *   Apply the same `enableMultiSort: false` and `enableSortingRemoval: false` configurations.

4.  **`src/app/planning/editorial-planning.tsx`**:
    *   Add `enableMultiSort: false` to disable sorting on multiple columns simultaneously.
    *   Add `enableSortingRemoval: false` to ensure that clicking a sorted column toggles only between ascending and descending (removing the "none" or unsorted state).
    *   **Confirm default sort**: The default sort for `Priority_Score` in descending order is already correctly set in the `initialState`.

### Icons for URL History:
Previously, I addressed the request to make all icons in the URL History pop-up (`src/features/shared/components/HistoryList.tsx`) use the primary color (`#00463c`). This change is already implemented.

### Implementation Details:
I will locate the `useReactTable` call in each of these files and add the specified configuration options to the options object passed to the hook. For example:

```typescript
  const table = useReactTable({
    data: ..., 
    columns: ..., 
    onSortingChange: setSorting,
    // ... other configurations
    enableMultiSort: false,          // Disable multi-column sorting
    enableSortingRemoval: false,     // Remove the 'none' sort state
    state: { sorting, ... },
  });
```

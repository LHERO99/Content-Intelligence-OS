# Feature Implementation: Keyword Ranking and UI Refinement

This plan outlines the integration of keyword ranking data and the refinement of the publishing workflow to ensure content integrity.

## 1. Database & Prioritization Changes

### A. Airtable & Types
- **Table**: `Keyword-Map` -> Add `Ranking` (Number).
- **Interface**: Update `KeywordMap` to include `Ranking?: number`.
- **Service**: Update `getKeywordMap`, `createKeyword`, and `updateKeyword` to handle the `Ranking` field.

### B. Striking Distance Scoring (`src/lib/prioritization-utils.ts`)
- **Logic**: 
    - 11-30: 100 pts (High priority).
    - 4-10: 40 pts.
    - 31-100: 60 pts.
    - 1-3: 10 pts.
    - Others: 30 pts.
- **Weights**: Add `weight_ranking` to prioritization weights and settings UI.

## 2. UI Refinements

### A. Ranking Visibility
- Add "Ranking" column to Keyword-Map, Suggestions, and Editorial Plan tables.
- Add Ranking input to `EditKeywordModal.tsx`.
- Support Ranking mapping in `KeywordImport.tsx`.

### B. Conditional Publishing Logic (`src/app/creation/ai-editor-workspace.tsx`)
- **Requirement**: Disable the **"Als veröffentlicht markieren"** button unless the status is **"Angeliefert"**.
- **Implementation**:
    - Pass the current keyword `Status` from `CreationPage` to `AIEditorWorkspace`.
    - Update button `disabled` logic: `disabled={status !== 'Angeliefert' || isSaving || isPublished}`.
    - Add a Tooltip to the button explaining the restriction when it's disabled.

## 3. Implementation Steps
1. **Update Types and Service**: Ensure core support for `Ranking`.
2. **Refactor Scoring**: Implement striking distance prioritization.
3. **Update Tables/Modals**: Add ranking visibility and editing.
4. **Restrict Publishing**: Implement the status-based button lock in the editor workspace.
5. **Verify**: Test that keywords in striking distance rise in priority, and the publish button only works for delivered content.

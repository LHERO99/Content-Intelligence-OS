# UI Refactoring: Dedicated Suggestions Tab and Planning Restructure

This plan outlines the final refactoring of the Planning UI to optimize the workflow and clean up unused sections.

## 1. UI Restructuring (Planning Section)

### A. Planning Page Layout (`src/app/planning/page.tsx`)
- **Tab Reordering & Cleanup**:
    - Reorder tabs to: **Redaktions-Planung** -> **Vorschläge** -> **Keyword-Map** -> **Blacklist**.
    - **Remove** the "Trend-Radar" tab and its associated component/import.
    - Set default tab to `editorial`.

### B. "Vorschläge" (Suggestions) Tab Optimization
- **Cleanup**: Remove the "Keywords importieren" (Import) button from this view.
- **Table Columns (`src/features/planning/components/suggestion-columns.tsx`)**:
    - Adjust columns to strictly include:
        1. **Keyword**
        2. **Typ** (`Action_Type`)
        3. **Aktion** (The "Hinzufügen" button)
        4. **Prio** (`Priority_Score`)
        5. **Suchvolumen**
        6. **Letzte Änderung** (Derived from `Last_Published` or latest log)
        7. **URL** (`Target_URL`)

### C. Keyword-Map Modal (`src/features/planning/components/EditEditorialModal.tsx` / `EditKeywordModal.tsx`)
- **Cleanup**: Remove the section labeled "Content-Plan Status" (or similar status badges/controls) from the keyword details popup to keep it focused on metadata.

# UI Logic Refinement: Suggestions Tab Filtering

This plan outlines the adjustment to the "Vorschläge" (Suggestions) tab to hide keywords once they have been published, ensuring they don't immediately reappear for optimization.

## 1. Requirement
Keywords that are marked as "Veröffentlicht" (Published) should be removed from the "Vorschläge" tab. They should only reappear after a specific period or when manually moved back to Backlog, to avoid cluttering the suggestions with fresh content.

## 2. Technical Implementation

### A. Filter Logic Update (`src/app/planning/suggestions-table.tsx`)
- Currently, the filter includes `kw.Status === 'Published'`.
- **Change**: Modify the `suggestionData` memo to **exclude** keywords with the status `'Published'`.
- **New Filter**:
    ```typescript
    const suggestionData = React.useMemo(() => {
      return keywords.filter(kw => 
        kw.Main_Keyword === 'Y' && 
        kw.Status === 'Backlog' // Remove 'Published' from allowed statuses
      );
    }, [keywords]);
    ```

## 3. Workflow Impact
- **New Content**: Appears in "Vorschläge" (Status: `Backlog`).
- **Planned/Active**: Moves to "Redaktionsplan" (Status: `Planned`, `Beauftragt`, etc.).
- **Published**: Disappears from **both** "Redaktionsplan" and "Vorschläge" (Status: `Published`).
- **Data Management**: The keyword remains accessible and visible in the full **Keyword-Map** tab for reference.

## 4. Implementation Steps
1. **Update `suggestions-table.tsx`**: Remove `'Published'` from the filter logic.
2. **Verify**: Publish a keyword and confirm it is no longer visible in the Suggestions tab.

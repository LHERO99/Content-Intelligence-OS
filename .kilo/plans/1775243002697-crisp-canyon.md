# UI Refactoring: Dedicated Suggestions Tab and Planning Restructure

This plan outlines the refactoring of the Planning UI to separate pure data management (Keyword-Map) from actionable planning (Vorschläge/Suggestions).

## 1. UI Restructuring (Planning Section)

### A. Keyword-Map Tab (`src/features/planning/components/keyword-columns.tsx`)
- **Remove "Content-Plan" Column**: The column with the "Hinzufügen" button and "Hinzugefügt" badge will be removed from the main Keyword-Map.
- **Purpose**: The Keyword-Map will now serve as a pure data repository for all keywords and their SEO metrics.

### B. New "Vorschläge" (Suggestions) Tab
- **Tab Component (`src/app/planning/suggestions-table.tsx`)**:
    - Implement a new table view specifically for actionable content planning.
    - **Filter Logic**:
        - Show only **Main Keywords** (`Main_Keyword === 'Y'`).
        - Show keywords with `Status === 'Backlog'` (New Content) or `Status === 'Published'` (Optimization candidates).
        - Support for future "Trend" topics from the `Potential_Trends` table.
- **Columns (`src/features/planning/components/suggestion-columns.tsx`)**:
    - Re-implement the "Hinzufügen" (Add to Editorial) button here.
    - Logic for `Action_Type` (Erstellung vs. Optimierung) remains as previously implemented.
    - Optically identical to the Keyword-Map for consistency.

### C. Planning Page Layout (`src/app/planning/page.tsx`)
- Update the `Tabs` component to include the new "Vorschläge" tab (using a `Sparkles` icon).
- Reorder tabs: **Vorschläge** -> **Redaktionsplan** -> **Keyword-Map**.

## 2. Component & Logic Changes

### A. Suggestions Logic
- **Recency Awareness**: The Suggestions tab will utilize the `Last_Published` date to show optimization candidates that haven't been updated for a certain period.
- **Status Sync**: When a keyword is added from the Suggestions tab, its status changes to `Planned`, and it moves from "Vorschläge" to the "Redaktionsplan".

### B. Publishing Sync
- Once a keyword is marked as "Veröffentlicht" in the Creation workspace:
    - Status becomes `Published`.
    - It disappears from "Redaktionsplan".
    - It reappears in "Vorschläge" as an optimization candidate (potentially with a lower priority score due to the recency factor).

## 3. Implementation Steps

1. **Create `suggestion-columns.tsx`**: Copy relevant columns from keyword-columns and include the `AddToEditorialButton`.
2. **Remove action columns from `keyword-columns.tsx`**.
3. **Create `suggestions-table.tsx`**: Component to fetch and filter keywords for the suggestions view.
4. **Update `src/app/planning/page.tsx`**: Add the new tab and integrate the `SuggestionsTable`.
5. **Verification**: Confirm that adding a keyword from "Vorschläge" moves it to "Redaktionsplan", and publishing it eventually brings it back to "Vorschläge".

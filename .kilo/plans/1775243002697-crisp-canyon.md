# Refactoring Plan: Status Logic and Content Cycle

This plan outlines the refactoring of status indicators and the content lifecycle across the Keyword-Map, Editorial Plan, and Creation Order List.

## 1. Database & Schema Changes (Airtable & Types)

### Standardize Status Options
We will standardize the `Status` field in the `Keyword-Map` table and `src/lib/airtable-types.ts` to use the following German/English hybrid (as currently partially implemented) or fully standardized set:
- `Backlog`: Default state in Keyword-Map.
- `Planned`: Added to Editorial Plan (corresponds to "Hinzugefügt" in Keyword-Map).
- `Beauftragt`: Commissioned via n8n in Editorial Plan.
- `In Arbeit`: Active creation phase (mapped from `Beauftragt` in the Order List).
- `Angeliefert`: Content received from n8n (replaces `Erstellt`).
- `Published`: Final state (replaces `Review` or similar final steps).

### Field Updates
- **Keyword-Map Table**: Ensure `Status` dropdown has these options.
- **Content-Log Table**: Ensure `Action_Type` supports `Erstellung` and `Optimierung`.

## 2. Component & Logic Changes

### A. Keyword-Map (`src/app/planning/page.tsx` & sub-components)
- **"Hinzufügen" Button**:
    - **Logic**: Only visible if `Status` is `Backlog` or `Published`.
    - **Action**: 
        - If `Status === 'Published'`, set `Action_Type` to `Optimierung` and `Status` to `Planned`.
        - If `Status === 'Backlog'`, set `Action_Type` to `Erstellung` and `Status` to `Planned`.
- **"Hinzugefügt" Badge**: Only show if `Status === 'Planned'`.
- **Visibility**: Once `Status` is `Published`, the entry shows the "Last Published" date and allows "Re-add".

### B. Editorial Plan (`src/app/planning/editorial-plan.tsx` & `src/app/api/n8n/trigger/route.ts`)
- **Filter**: Shows keywords where `Status === 'Planned'`.
- **Commissioning**: 
    - When "Beauftragen" is clicked, the API trigger sets `Status` to `Beauftragt`.
    - The entry remains in the Editorial Plan but shows status `Beauftragt`.

### C. Creation Order List (`src/app/creation/page.tsx`)
- **Filter**: Shows keywords where `Status` is `Beauftragt`, `In Arbeit`, or `Angeliefert`.
- **Status Display Logic**:
    - `Beauftragt` OR `In Arbeit` → Display as **"In Arbeit"**.
    - `Angeliefert` → Display as **"Angeliefert"**.
- **Publishing Action**:
    - Implement a "Veröffentlichen" button in the editor/workspace.
    - Clicking this sets `Status` to `Published`.
    - **Result**: The entry automatically disappears from the Order List and Editorial Plan (due to filters) and resets in the Keyword-Map.

### D. API Routes
- **n8n Callback (`src/app/api/n8n/callback/route.ts`)**:
    - Update `Status` to `Angeliefert` upon successful content delivery.
- **Keyword Updates (`src/app/api/planning/keywords/route.ts`)**:
    - Ensure status transitions handle the `Action_Type` logic (Erstellung vs. Optimierung).

## 3. Implementation Steps

1. **Update Types**: Modify `KeywordStatus` in `src/lib/airtable-types.ts`.
2. **Update API (n8n)**: Adjust trigger and callback routes for `Beauftragt` and `Angeliefert`.
3. **Update Keyword-Map UI**: Refactor "Add" button logic and status badges.
4. **Update Editorial Plan UI**: Ensure "Beauftragt" reflects correctly.
5. **Update Creation UI**: 
    - Standardize status display in the list.
    - Implement the "Veröffentlichen" button/action.
6. **Verification**: Test the full cycle from Backlog -> Planned -> Beauftragt -> In Arbeit -> Angeliefert -> Published -> Re-add (Optimierung).

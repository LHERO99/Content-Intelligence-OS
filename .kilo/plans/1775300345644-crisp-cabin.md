# Plan: Comprehensive Logging Refactor & Performance Fix

The user reports that logging is still missing for three critical steps:
1.  **URL added to Keyword-Map:** "URL der Keyword-Map hinzugefügt" (Import or manual creation).
2.  **URL added to Proposal List:** "URL der Vorschlags-Liste hinzugefügt" (Automatic base on status).
3.  **Content commissioned:** "Content beauftragt" (Status change to 'Beauftragt').

## Root Cause Analysis
- **Bulk Import/Creation:** The `bulkCreateKeywords` logic in `src/lib/airtable.ts` was missing the `Target_URL` field in the log entries. Since history views filter by `Target_URL`, these logs were invisible.
- **Single Creation:** Similar issue where `Target_URL` wasn't always reliably passed or mapped.
- **Commissioning:** The "Content beauftragt" log was manually handled in `/api/n8n/trigger`, making it inconsistent if the status was changed elsewhere.

## Proposed Changes

### 1. Centralize All Initial & Status Logging in `src/lib/airtable.ts`
- **`bulkCreateKeywords`:** Fix the log creation loop to include `Target_URL`.
- **`createKeyword`:** Ensure `Target_URL` is passed to `createContentLog`.
- **`updateKeyword`:** Add automatic logging for status transitions:
    - If `Status` changes to `Backlog` -> Log "URL der Vorschlagsliste hinzugefügt".
    - If `Status` changes to `Beauftragt` -> Log "Content beauftragt".
    - If `Status` changes to `Planned` -> Log "URL der Redaktionsplanung hinzugefügt".

### 2. Remove Redundant Logging from API Routes
- Remove manual `createContentLog` calls from:
    - `src/app/api/planning/import/route.ts`
    - `src/app/api/planning/keywords/route.ts` (POST and PATCH)
    - `src/app/api/n8n/trigger/route.ts` (The "Content beauftragt" log)
- This prevents duplicates and ensures logging is consistent across all entry points (UI, API, N8N).

### 3. Performance & Reliability
- Keep the chunked bulk log creation in `bulkCreateKeywords` for speed.
- Wrap all service-level logging in `try-catch` to ensure core data operations (creating/updating keywords) never fail due to logging errors.

## Implementation Details

### `src/lib/airtable.ts`
- **`updateKeyword`:** Will now detect status changes and trigger the appropriate log.
- **`bulkCreateKeywords`:** Will include `Target_URL` in its chunked log creation.

3.  **Commissioning:** Click "Content beauftragen" and verify the "Content beauftragt" log appears in the history.

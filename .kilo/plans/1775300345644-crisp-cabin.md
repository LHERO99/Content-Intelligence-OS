# Plan: Simplified Content History for Planning Modals

The user wants a simplified display of the content history in the planning pop-ups, showing only the last time content was created or optimized for the URL.

## Current State
- `EditKeywordModal.tsx` and `EditEditorialModal.tsx` use the `HistoryList` component, which shows a full list of all "Nahrungskette" events.
- `HistoryList.tsx` filters logs based on a predefined list of action types.

## Proposed Changes

### 1. Create a New Component `LastActionHistory.tsx`
- Create a new shared component in `src/features/shared/components/LastActionHistory.tsx`.
- This component will:
  - Take `history` and `isLoading` as props (same as `HistoryList`).
  - Filter the history for `Diff_Summary` matching "Content veröffentlicht" (which represents both creation and optimization depending on the `Action_Type` in the log).
  - Alternatively, filter for specific summaries if the user wants to see specifically "Content erstellt" or "Content optimiert" labels.
  - Display a concise message like: "Zuletzt optimiert am 04.04.26" or "Zuletzt erstellt am 01.03.26".
  - If no such entry exists, show "Noch kein Content veröffentlicht".

### 2. Update Planning Modals
- Replace `HistoryList` with `LastActionHistory` in:
  - `src/features/planning/components/EditKeywordModal.tsx`
  - `src/features/planning/components/EditEditorialModal.tsx`

### 3. Logic for "Zuletzt erstellt/optimiert"
In `LastActionHistory.tsx`:
```typescript
const lastPublishedLog = history.find(log => log.Diff_Summary === "Content veröffentlicht");

// If Action_Type is stored in the log, use it to distinguish
const label = lastPublishedLog?.Action_Type === 'Optimierung' ? 'Zuletzt optimiert' : 'Zuletzt erstellt';
const date = lastPublishedLog ? formatDate(lastPublishedLog.Created_At) : null;
```

## Implementation Details

### Shared Components (`src/features/shared/components/index.ts`)
- Export the new `LastActionHistory` component.

### Planning Modals
Update the JSX:
```tsx

```

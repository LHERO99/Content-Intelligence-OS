# Plan: Fix Missing Content Logs for Initial URL Creation/Import

The user reports that logging is again missing for:
1.  **URL Import/Addition to Keyword-Map:** "URL der Keyword-Map hinzugefügt" is missing.
2.  **Addition to Proposals Tab:** "URL der Vorschlagsliste hinzugefügt" is missing.

## Current Investigation
- `bulkCreateKeywords` in `src/lib/airtable.ts` has logic to create logs in parallel batches *after* the records are created.
- However, the log creation uses `base(TABLES.CONTENT_LOG).create(chunk.map(...))` where it only passes `Keyword_ID`, `Diff_Summary`, and `Action_Type`.
- **CRITICAL MISSING FIELD:** The `Target_URL` field is missing in the bulk log creation inside `bulkCreateKeywords`. Since the history list and details views often filter or fetch by `Target_URL`, logs without this field won't show up in the specific URL's history.
- `createKeyword` (single) also has logging logic, but it uses `Keyword_ID` (array) and `Target_URL`.

## Proposed Changes

### 1. Fix `bulkCreateKeywords` in `src/lib/airtable.ts`
- Update the bulk log creation to include the `Target_URL` field for each log entry.
- Ensure the `Target_URL` is correctly mapped from the `createdRecords`.

### 2. Fix `createKeyword` in `src/lib/airtable.ts`
- Ensure `Target_URL` is passed correctly to `createContentLog`.
- Double check if `createContentLog` itself handles `Target_URL` correctly (it does, but `bulkCreateKeywords` was manually calling `base().create()` bypassing `createContentLog`'s mapping).

### 3. Verify `createContentLog` Mapping
- In `createContentLog`, the mapping for `Target_URL` might be problematic if Airtable treats it as a lookup field (which it often is in the Content-Log table pointing to Keyword-Map).
- If `Target_URL` in `Content-Log` is a formula or lookup, we shouldn't (and can't) write to it. If it's a text field used for filtering, we must write to it.
- Based on `getContentHistoryByUrl`, it uses `{Target_URL} = '${targetUrl}'`, implying it's a filterable field.

## Implementation Details

### `src/lib/airtable.ts`
Modify the bulk log creation loop:
```typescript
await base(TABLES.CONTENT_LOG).create(
  chunk.map(log => ({
    fields: {
      Keyword_ID: log.Keyword_ID,
      Target_URL: log.Target_URL, // ADD THIS
      Diff_Summary: log.Diff_Summary,
      Action_Type: log.Action_Type,
    }
  }))
);
```

# Plan for Logging Blacklisted and Restored URLs

## Problem
Currently, when a URL is blacklisted, it is logged. However, when a URL is restored from the blacklist, the logging needs to be updated to ensure the history is consistent and properly reflects that the URL is no longer blacklisted.

## Analysis
The restoration process happens in `src/app/planning/blacklist.tsx` within the `restoreEntry` function. It performs two steps:
1.  Creates a new keyword in the `Keyword-Map` via `POST /api/planning/keywords`.
2.  Deletes the entry from the `Blacklist` via `DELETE /api/planning/blacklist`.

Currently, neither of these steps (or the frontend logic) creates a `Content-Log` entry to mark the restoration.

## Proposed Solution

1.  **Modify `src/app/planning/blacklist.tsx`**: Update the `restoreEntry` function to create a content log entry when a URL/Keyword is successfully restored. This log should include the `Logged_URL` so it appears correctly in the history.

2.  **Ensure Consistency**: The log should use a clear `Diff_Summary` like "URL von der Blacklist entfernt" or "Eintrag wiederhergestellt".

## Detailed Steps

1.  **Modify `src/app/planning/blacklist.tsx`**:
    *   In the `restoreEntry` function, after the keyword is successfully created in the `Keyword-Map` (after `createResponse.ok` check), add a call to create a content log.
    *   Since there isn't a dedicated "restore" API that logs, we can either:
        *   Call `POST /api/planning/history` directly from the frontend.
        *   Update `/api/planning/keywords` POST handler to accept a flag or automatically log if it's a "restoration" (less clean as the API doesn't know it's a restore).
        *   **Recommendation**: Call `POST /api/planning/history` from `restoreEntry` in `blacklist.tsx`.

2.  **Implementation Details for `restoreEntry`**:
    ```typescript
    // After successful keyword creation
    const restoredKeyword = await createResponse.json();
    await fetch("/api/planning/history", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        keywordId: restoredKeyword.id,
        url: formData.Target_URL,
        Logged_URL: formData.Target_URL, // Ensure our new field is set
        Action_Type: restoredKeyword.Action_Type || 'Optimierung',
        Diff_Summary: 'URL von der Blacklist entfernt',
        Reasoning_Chain: `Eintrag wurde aus der Blacklist wiederhergestellt.\nUrsprünglicher Eintrag: ${entry.Keyword}`,
      }),
    });
    ```

## Verification Steps
1.  Add a URL to the blacklist.
2.  Verify it shows "Blacklisted" in `/history`.
3.  Restore the URL from the Blacklist page.
4.  Navigate to `/history` and verify:
    *   The URL no longer has the "Blacklisted" badge (or has a "Restored" indicator if we implement one).
    *   A new entry "URL von der Blacklist entfernt" is visible in the history.
    *   Previous history is still visible.

# Plan: Fix Server Error when Adding Keyword to Content Plan

The user reports a server error (with a `url.parse()` deprecation warning) when clicking "Hinzufügen" in the Keyword-Map popup.

## Root Cause Analysis
Based on the code analysis:
1. **API Route Vulnerability**: The `PATCH` handler in `src/app/api/planning/keywords/route.ts` lacks proper error handling for the `createContentLog` call. If Airtable fails to create the history log (e.g., due to schema mismatch), the entire request fails with a 500 error.
2. **Missing Record Check**: It assumes `currentKeyword` is found. If it's not, it might crash later when accessing `currentKeyword.Status`.
3. **Deprecation Warning**: The `url.parse()` warning is likely coming from the `airtable` library itself or Next.js internals when handling the API request, but it's not the cause of the 500 error.

## Proposed Changes

### 1. API Route Hardening (`src/app/api/planning/keywords/route.ts`)
- Add a check to verify if the keyword exists before attempting an update.
- Wrap the `createContentLog` call in a `try...catch` block to ensure that a logging failure doesn't crash the main update operation.
- Add detailed server-side logging (`console.log`) to the `PATCH` handler to track incoming data and trace where it fails.

### 2. Frontend Check (`src/features/planning/components/EditKeywordModal.tsx`)
- Verify that the `onSave` function correctly handles the updated `Planned` status. (Already done in previous steps, but will double-check).

### 3. Service Verification (`src/features/planning/services/planning-service.ts`)
- Ensure `updateKeyword` is sending the JSON body correctly. (Already done in previous steps).

## Step-by-Step Execution Plan
1. **Modify API Route**: Update `src/app/api/planning/keywords/route.ts` to include record existence checks and safe logging.
2. **Add Logging**: Add more context to the server logs to see the actual error message from Airtable if the 500 occurs.
3. **Verification**: Request the user to try again and provide logs if it still fails.

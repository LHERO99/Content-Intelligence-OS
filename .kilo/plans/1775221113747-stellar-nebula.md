# Implementation Plan: Comprehensive Content Workspace & UI Fixes

## Goal
Resolve all remaining issues in the Content Creation and Planning workspace, focusing on visual consistency, formatting, and UX stability.

## Tasks

### 1. Editorial Planning: "Commission" Button Stability
- **Issue**: The "Beauftragen" button reappears after a short time.
- **Fix**: Update `src/app/planning/editorial-planning.tsx` to hide the button based on a comprehensive set of statuses (`In Progress`, `Beauftragt`, `In Arbeit`, `Erstellt`, `Review`, `Optimierung`) in addition to history checks.

### 2. Content Creation: Aufträge List Highlighting
- **Issue**: Selected items (especially those with content) don't consistently show the green active indicator.
- **Fix**: Update `src/app/creation/page.tsx` to use high-priority CSS classes (`!important`, `inset shadow`) for the active row state regardless of the item's status.

### 3. Content Creation: Preview Tab Formatting
- **Issue**: Headlines and lists appear as plain text in the "Vorschau" tab.
- **Fix**: Use `global` JSX styles in `src/app/creation/ai-editor-workspace.tsx` to target dynamically injected HTML.
- **Style Specs**:
    - **Font**: Poppins.
    - **Headlines (H1-H3)**: Color #00463c, Font-weight 800, Line-height 1.2.
    - **Margins**: H1 margin-top: 0, others 1.25rem top / 0.5rem bottom.
    - **Lists**: Visible bullets (disc/decimal) and indentation.
    - **Padding**: Set container padding to `p-3`.

### 4. Content Creation: Editor Styling
- **Padding**: Set editor (`ProseMirror`) padding to `0.75rem 2rem`.
- **Headline Margins**: Sync margins with preview (1.25rem top / 0.5rem bottom).
- **Lists**: Ensure lists are visually formatted in the editor.

### 5. Modals: Layout & Error Handling
- **Keyword Map Popup**: Fix overflow of long URLs in error alerts using `break-words` and `overflow-hidden`.
- **Importer Modal**: Fix layout of the "skipped records" box. Add `mx-6`, `overflow-hidden`, and ensure the list items inside `ScrollArea` are truncated/bounded.

### 6. Workspace Synchronization
- **HTML/Visual Sync**: Ensure `codeContent` and Tiptap editor are reliably synced when switching modes and saving.
- **Immediate Save**: Ensure "Speichern" triggers a global refresh event.

### 8. Modal UI: Standardized Content History
- **Issue**: The "Content-Historie" section in the Keyword-Map modal has a different layout compared to the Editorial Planning modal.
- **Action**:
    - Update the `EditKeywordModal` component in `src/app/planning/keyword-table.tsx`.
    - Replace the existing history list implementation with the layout from `EditEditorialModal` in `src/app/planning/editorial-planning.tsx`.
    - This includes the styling for the "Latest Action Highlight" (p-3, rounded-lg, bg-[#00463c]/5) and the icon/text structure for history entries.
- Test button persistence in Editorial Planning.
- Verify consistent row highlighting in Aufträge list.
- Check visual rendering of H1-H3 and lists in the Preview tab (Poppins font).
- Confirm error messages in popups wrap correctly and don't break the UI.

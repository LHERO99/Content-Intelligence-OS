# Implementation Plan: Content Workspace Fixes & Sync (Final)

## Goal
Fix visual highlighting, formatting in preview, editor spacing, and the "Commission" button persistence issue in the editorial planning view.

## Tasks

### 1. Fix "Commission" Button Persistence
- **Problem**: After clicking "Beauftragen", the button reappears shortly after, even though the status should be "In Progress" or "Beauftragt".
- **Analysis**: 
    - The `EditorialPlanning` component uses a local `commissionedIds` state that is updated immediately, but it is overwritten by the next polling cycle (`fetchGlobalHistory`) if the Airtable record hasn't fully processed the new `Content-Log` entry.
    - The logic for showing the button `!commissionedIds.has(id)` depends on the existence of a log entry. 
- **Action**: 
    - Modify `src/app/planning/editorial-planning.tsx` to also check the keyword's `Status` field. If `Status === 'In Progress'`, 'Beauftragt', or 'In Arbeit', the button should NOT be shown, even if the history log is still being created.
    - Ensure the local `commissionedIds` state persists or is merged correctly with the incoming polling data to prevent flickering.

### 2. Fix Visual Highlighting & Formatting (Verify/Finalize)
- **Aufträge List**: Ensure `!important` styles and inset shadows are applied to the active row.
- **Preview Tab**: Ensure `global` JSX styles are used to format dynamically injected HTML (Headlines, Lists).
- **Editor**: Verify headline margins (1.25rem top, 0.5rem bottom).

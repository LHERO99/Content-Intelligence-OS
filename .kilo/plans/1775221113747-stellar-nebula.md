# Implementation Plan: Content Workspace Fixes (Revised)

## Goal
Fix commission timestamp formatting, integrate HTML source view directly into the "Edit" tab, resolve CSS/Editor issues preventing H-tags from being visually distinct, refine UI colors, and fix synchronization issues between HTML and visual editor modes.

## Tasks

### 1. Fix HTML to Visual Editor Synchronization
- **Issue**: Changes made in "Code View" (HTML) are not immediately reflected when switching back to the visual editor or when saving.
- **Analysis**: The `useEffect` in `RichTextEditor` that updates the Tiptap content from `codeContent` only triggers when `showCode` becomes `false`. However, the Tiptap instance needs a more explicit update when the user switches back.
- **Action**: 
    - Ensure `editor.commands.setContent(codeContent)` is called correctly and reliably when switching from Code to Editor mode.
    - Ensure `onSave(codeContent)` is called when in Code mode, and that this update propagates back to the `AIEditorWorkspace` state.
    - Add a `onUpdate` or similar listener to Tiptap to keep `codeContent` in sync even while typing in visual mode, or ensure the final state is captured before switching/saving.

### 2. UI Color Refinement (Verify)
- Commission Timestamp: Neutral muted color.
- Action Type Badge: Subtle gray.

### 3. Editor Performance & HTML Beautification (Verify)
- **Active State**: Instant response for H1-H3 buttons.
- **Paragraph Toggle**: "Pilcrow" button added.
- **HTML Beautification**: `js-beautify` integrated for structured code view.

### 4. Fix Visual Formatting (H-Tags & Paragraphs) (Verify)
- Ensure H1, H2, H3, and P tags have clear visual distinction and margins in the editor.

## Verification
- Edit text in "Code View", switch to "Editor" -> changes must be visible.

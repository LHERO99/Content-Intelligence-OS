# Implementation Plan: Content Workspace Fixes & Styling (Final)

## Goal
Fix visual highlighting in the commission list, resolve formatting issues in the preview tab, and adjust headline margins in the editor.

## Tasks

### 1. Fix Visual Highlighting in "Aufträge" List
- **Problem**: Selected items with delivered content (Status 'Erstellt' etc.) don't show the active visual state (green left border).
- **Analysis**: The `TableRow` styling in `src/app/creation/page.tsx` is being overwritten by table-level or badge-level styles.
- **Action**: Use a more specific and forced selector for the active row. Ensure the `selectedKeywordId === kw.id` logic applies consistent classes regardless of the item's status. Add `!border-l-emerald-600` and ensure the background color is forced.

### 2. Fix Formatting in Preview Tab
- **Problem**: Headlines (H1, H2, H3) are visually indistinguishable in the "Vorschau" tab despite correct HTML tags.
- **Analysis**: The Tailwind `prose` class might not be working correctly due to nested divs or missing global styles for headings within the preview container.
- **Action**: 
    - In `src/app/creation/ai-editor-workspace.tsx`, ensure the preview container has a clear, non-conflicting structure.
    - Inject specific `<style>` overrides for the preview container if `prose` continues to fail, ensuring `H1`, `H2`, and `H3` have defined sizes and weights.

### 3. Adjust Editor Headline Margins
- **Problem**: Headline margins in the editor need adjustment.
- **Action**: Update `src/app/creation/rich-text-editor.tsx`. In the `.ProseMirror` global style block, set:
    - `margin-top: 1.25rem !important`
    - `margin-bottom: 0.5rem !important`
    - Apply this to `h1`, `h2`, and `h3`.

## Verification
- Select an "Erstellt" entry: Green border must appear.

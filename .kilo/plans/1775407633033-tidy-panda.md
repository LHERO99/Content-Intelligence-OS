# Plan for Fixing Missing Logging for Keyword-Map and Suggestions Tab

## Problem
Currently, adding a single keyword (via `src/app/api/planning/keywords/route.ts`) or a trend (via `src/app/api/monitoring/suggest/route.ts`) does not trigger the appropriate content logs. Specifically:
1.  **Keyword Addition**: When a single keyword is created via the API, the "URL wurde dem Tool hinzugefügt" and "URL wurde dem Tab 'Vorschläge' hinzugefügt" (if applicable) logs are missing.
2.  **Trend Addition**: When a trend is suggested from monitoring, there is a placeholder comment but no actual logging to `Content-Log`.

## Analysis
-   **`src/app/api/planning/keywords/route.ts` (POST)**: Calls `createKeyword` but does not follow up with `createContentLog`. The bulk import API (`import/route.ts`) already has this logic, so it should be mirrored here.
-   **`src/app/api/monitoring/suggest/route.ts` (POST)**: Creates a record in `Potential_Trends`. While trends are not yet in `Keyword-Map`, the user expects a log entry for the URL indicating it was added to "Vorschläge" (Suggestions).
-   **`src/app/api/planning/trends/route.ts` (POST)**: Similar to the suggest API, it creates a trend but lacks logging.

## Proposed Solution

### 1. Update Keyword POST API
Add logging logic to `src/app/api/planning/keywords/route.ts` after successful creation:
-   Log: "URL wurde dem Tool hinzugefügt"
-   Conditional Log: "URL wurde dem Tab 'Vorschläge' hinzugefügt" (if `Status === 'Backlog' && Main_Keyword === 'Y'`)

### 2. Update Monitoring Suggest API
Add logging logic to `src/app/api/monitoring/suggest/route.ts`:
-   Log: "URL wurde dem Tab 'Vorschläge' hinzugefügt"
-   *Note*: Since `Potential_Trends` records don't have a `Keyword_ID` compatible with `Content-Log`'s link field, we must rely on `Logged_URL` and `Target_URL` (string) for these logs.

### 3. Update Planning Trends API
Add logging logic to `src/app/api/planning/trends/route.ts` to ensure manual trend creation is also logged.

## Detailed Steps

### Step 1: Modify `src/app/api/planning/keywords/route.ts`
-   Import `getServerSession` and `authOptions` (already present).
-   After `const result = await createKeyword(...)`:
    -   Fetch session.
    -   Call `createContentLog` for "URL wurde dem Tool hinzugefügt".
    -   If `result.Status === 'Backlog' && result.Main_Keyword === 'Y'`, call `createContentLog` for "URL wurde dem Tab 'Vorschläge' hinzugefügt".

### Step 2: Modify `src/app/api/monitoring/suggest/route.ts`
-   After `const trend = await createTrend(...)`:
    -   Call `createContentLog` with `Logged_URL: url` and `Diff_Summary: "URL wurde dem Tab 'Vorschläge' hinzugefügt"`.
    -   *Constraint*: `Keyword_ID` will be empty/undefined as it's not a Keyword-Map record yet.

### Step 3: Modify `src/app/api/planning/trends/route.ts`
-   Import `getServerSession`, `authOptions`, and `createContentLog`.
-   After successful trend creation:
    -   Log "URL wurde dem Tab 'Vorschläge' hinzugefügt" if a URL is associated (parsed from topic or explicitly available).

## Verification Plan
1.  **Manual Keyword Creation**:
    -   Add a keyword via the UI (Planning -> Add Keyword).
    -   Check `/history` for the URL.
    -   Verify "URL wurde dem Tool hinzugefügt" and "URL wurde dem Tab 'Vorschläge' hinzugefügt" appear.
2.  **Monitoring Suggestion**:
    -   Trigger "Submit to Suggestions" from the Monitoring page.
    -   Check `/history` for the URL.
    -   Verify "URL wurde dem Tab 'Vorschläge' hinzugefügt" appears.
3.  **Trend Creation**:
    -   Add a trend manually.
    -   Verify logging.

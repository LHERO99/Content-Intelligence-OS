# Implementation Plan - Display commissioned keywords in "Content-Erstellung"

The user wants to ensure that all keywords commissioned from the editorial plan (Redaktionsplan) are visible in the "Content-Erstellung" page's task list. Currently, the filtering logic for `commissionedKeywords` might be too restrictive or inconsistent with how "commissioning" is actually implemented in the system.

## Analysis
- **Commissioning Trigger**: In `src/app/api/n8n/trigger/route.ts`, when `COMMISSION_CONTENT` is triggered:
  - The keyword status is updated to `In Progress`.
  - A `Content-Log` entry is created with `Action_Type: 'Erstellung'` and `Diff_Summary: 'Content beauftragt'`.
- **Current Filter in `CreationPage`**:
  - Checks if status is `Beauftragt` or `In Progress`.
  - OR checks if there is any history entry in `contentLogs`.
- **Potential Issue**: 
  - The `Status` labels in the UI/Airtable might have changed (e.g., German vs English).
  - The `contentLogs` used for filtering might be limited by the `maxRecords: 100` in `getContentLogs`.
  - The `Action_Type` in `Content-Log` might be important to distinguish between "commissioned" and "just edited".

## Proposed Changes

### 1. Robust Filtering Logic in `CreationPage`
Refine the `commissionedKeywords` filter to be more inclusive and resilient to status label variations.

### 2. Status Mapping
Ensure that "In Progress" (technical status) is correctly mapped to the German UI label "In Arbeit" or "Beauftragt".

### 3. Data Fetching
Ensure that all relevant keywords with active workflows are fetched, even if they aren't in the first 100 history entries.

## Implementation Steps

### Task 1: Update `src/app/creation/page.tsx`
- Modify the `commissionedKeywords` filter to include all keywords that are part of the editorial cycle.
- Add logging to debug why certain keywords might be missing during development.
- Ensure the "Status" badge colors and labels match the actual Airtable values.

### Task 2: Verification
- Check the `Editorial Planning` module to see what status it sets when "Beauftragen" is clicked.
- Verify that keywords set to "In Progress" appear immediately in the "Content-Erstellung" list due to the 5s polling.

## Technical Details
- **File**: `src/app/creation/page.tsx`
- **Logic**: 
  ```typescript
  const commissionedKeywords = keywords.filter(kw => {
    // Include if status is part of the creation pipeline
    const isActiveStatus = ['Beauftragt', 'In Progress', 'In Arbeit', 'Review'].includes(kw.Status);
    
    // Also include if it has any history entry (fallback)
    const hasHistory = contentLogs.some(l => 
      Array.isArray(l.Keyword_ID) && l.Keyword_ID.includes(kw.id)
    );

    return isActiveStatus || hasHistory;
  });
  ```

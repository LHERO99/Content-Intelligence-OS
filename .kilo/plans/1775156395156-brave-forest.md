# Implementation Plan - Enhanced Content Action Workspace

The goal is to evolve the "Content-Erstellung" workspace into a fully interactive tool with three primary actions: Manual Editing (HTML), AI-driven Editing (Chat), and Forwarding to Pharma (Action Trigger).

## Analysis
- **Current State**: `AIEditorWorkspace` displays either a static preview or a diff view. No editing capability.
- **Requirement 1: Manual Edit**: Integration of an HTML editor (e.g., Tiptap) to allow formatting, headlines, and links.
- **Requirement 2: AI Edit**: A side-panel or overlay chat interface to interact with n8n/LLM for contextual text transformations.
- **Requirement 3: Pharma Forward**: A UI button for future integration.

## Proposed Changes

### 1. Component Architecture Update
- Refactor `AIEditorWorkspace` to include a toolbar for switching between "View", "Edit", and "AI Chat" modes.
- Implement a state machine for the content: `original` (v1), `proposed` (v2), and `working` (active edits).

### 2. Integration of HTML Editor
- Recommended: **Tiptap** (modern, headless, React-friendly).
- Toolbar needs: H1-H3, Bold, Italic, Link, List, Save.
- Storage: Saving should trigger a POST to `/api/planning/history` with a new version/log entry.

### 3. AI Chat Interface
- New component: `AIChatPanel`.
- UI: Message list + input field.
- Integration: Sends prompts to `/api/n8n/trigger` (Action: `REFINE_CONTENT`).
- Logic: The chat should receive the *current working text* as context and return the *optimized text*.

### 4. Action Toolbar
- Add a persistent action bar above or below the content:
    - [Bearbeiten] -> Switches to Editor.
    - [KI-Optimierung] -> Opens Chat.
    - [An Pharma senden] -> Branded primary action (UI only for now).

## Implementation Steps

### Task 1: Scaffolding & Mode Switching
- Add `Tiptap` dependencies (`@tiptap/react`, `@tiptap/starter-kit`, etc.).
- Update `AIEditorWorkspace` props and internal mode state.

### Task 2: HTML Editor Integration
- Implement `RichTextEditor` using Tiptap.
- Add "Speichern" logic to persist edits to Airtable via `Content-Log`.

### Task 3: AI Chat Panel
- Create the chat UI in a side drawer or integrated column.
- Wire up the n8n trigger for content refinement.

### Task 4: Branded Actions & Pharma Button
- Add the "An Pharma senden" button with DocMorris branding.
- Finalize the layout for seamless transitions between modes.

## Technical Details
- **Persistence**: Edits should be saved as a new entry in `Content-Log` with `Version: 'v2-edited'` or similar to maintain history.
- **n8n Action**: `action: 'REFINE_CONTENT'` with payload `{ currentContent, instructions, keywordId }`.

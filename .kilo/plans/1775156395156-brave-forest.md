# Implementation Plan - Refine KI-Optimierung UX

The goal is to improve the usability of the AI optimization tab by fixing the chat panel height and removing unnecessary UI elements.

## Analysis
- **Issue 1: Chat Window Height**: The chat window currently grows with the content, which is bad for usability. It should have a fixed, consistent height within its container.
- **Issue 2: Reasoning Chain**: The user wants to remove the "Reasoning Chain" panel from the creation page.
- **Components involved**:
    - `src/app/creation/ai-chat-panel.tsx`: Controls the chat UI.
    - `src/app/creation/ai-editor-workspace.tsx`: Hosts the chat panel.
    - `src/app/creation/page.tsx`: Displays the overall layout including the reasoning panel.

## Proposed Changes

### 1. Fix Chat Usability (`src/app/creation/ai-chat-panel.tsx`)
- Ensure the `AIChatPanel` uses a fixed height layout where only the message area is scrollable.
- The parent container in `AIEditorWorkspace` already sets `h-[600px]`, but we should ensure the chat panel occupies this space correctly without pushing the UI.

### 2. Remove Reasoning Chain (`src/app/creation/page.tsx`)
- Remove the `<ReasoningPanel />` component from the `CreationPage`.
- Adjust the layout to reclaim the space previously occupied by the reasoning panel.

### 3. Layout Optimization (`src/app/creation/ai-editor-workspace.tsx`)
- Refine the grid layout in `ai-chat` mode to ensure the "Aktueller Arbeitsstand" and "AIChatPanel" are perfectly aligned and have a stable height.

## Implementation Steps

### Task 1: Clean up `page.tsx`
- Remove the reasoning panel and its associated grid container.

### Task 2: Refine Chat Panel Height
- Update `AIChatPanel` to ensure it fills its container's height properly.
- Update `AIEditorWorkspace` to use a more stable height for the optimization view.

### Task 3: Verification
- Verify that the chat area has a fixed height and only messages scroll.
- Verify that the reasoning chain is gone.

# Implementation Plan - Redaktionsplanung Extension (Type Field)

The goal is to add a "Typ" field (Erstellung vs. Optimierung) to the "Redaktions-Planung" table and allow editing/managing this field during manual entry and keyword editing.

## Analysis
- **Data Model**: The `KeywordMap` interface in `src/lib/airtable-types.ts` needs a new field `Action_Type` (or similar, though `ContentLog` already uses `Action_Type`). To keep it consistent with the "Content-Erstellung" logic, we should add a field to the `Keyword-Map` table in Airtable called `Action_Type` with values `'Erstellung' | 'Optimierung'`.
- **UI Components**:
    - `src/app/planning/editorial-planning.tsx`: Add the "Typ" column after the "Status" column.
    - `src/app/planning/keyword-table.tsx`: Add the "Typ" column for consistency in the Keyword-Map view.
    - `src/app/planning/add-entry-fab.tsx`: Add a dropdown to select the type when creating a new keyword.
    - `src/app/planning/editorial-planning.tsx` & `src/app/planning/keyword-table.tsx`: Update the edit modals to include the type selection.

## Proposed Changes

### 1. Update Type Definitions (`src/lib/airtable-types.ts`)
- Add `Action_Type?: 'Erstellung' | 'Optimierung';` to the `KeywordMap` interface.

### 2. Update Airtable Integration (`src/lib/airtable.ts`)
- Update `getKeywordMap`, `createKeyword`, and `updateKeyword` functions to handle the new `Action_Type` field.

### 3. Update Manual Entry (`src/app/planning/add-entry-fab.tsx`)
- Add a new state `actionType` (defaulting to `'Erstellung'`).
- Add a UI `Select` field in the "keyword" creation form.
- Include `Action_Type: actionType` in the POST body.

### 4. Update Table Views
- **Editorial Planning** (`src/app/planning/editorial-planning.tsx`):
    - Add `Action_Type` to `columns` definition.
    - Place it after the `Status` column.
    - Use a `Badge` for display (similar to the one in Content-Erstellung).
    - Update `EditEditorialModal` to include a `Select` for the type.
- **Keyword Map** (`src/app/planning/keyword-table.tsx`):
    - Add `Action_Type` to `columns` definition.
    - Update `EditKeywordModal` to include a `Select` for the type.

## Implementation Steps

### Task 1: Data Layer
- Update `airtable-types.ts` and `airtable.ts`.

### Task 2: Entry & Edit UI
- Update `add-entry-fab.tsx` with the new field.
- Update `EditKeywordModal` in `keyword-table.tsx` and `EditEditorialModal` in `editorial-planning.tsx`.

### Task 3: Table Presentation
- Update the `columns` array in both table files to include and display the new column.

### Task 4: Verification
- Verify that the type is correctly saved to Airtable.
- Verify that selecting "Optimierung" in the planning table results in the "Diff View" being shown in the "Content-Erstellung" page (as implemented in the previous task).

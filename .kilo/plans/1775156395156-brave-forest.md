# Implementation Plan - Content Creation Type Expansion

The goal is to differentiate between "Erstellung" (Creation) and "Optimierung" (Optimization) in the "Content-Erstellung" view. The UI will show the type in the task list and adapt the editor view based on the type (Single View vs. Diff View).

## Analysis
- **Data Model**: The `ContentLog` already has an `Action_Type` field with values `'Planung' | 'Erstellung' | 'Optimierung'`.
- **Creation Page**: Currently always shows a `ReactDiffViewer` in `AIEditorWorkspace`.
- **Logic**:
    - If `Action_Type === 'Erstellung'`: Show only the generated content (v2).
    - If `Action_Type === 'Optimierung'`: Show Diff View (v1 vs v2).

## Proposed Changes

### 1. Update `src/app/creation/page.tsx`
- In the task list (Left Side), add a badge or text indicating the "Typ" (Erstellung or Optimierung).
- Pass the `Action_Type` of the selected keyword's latest log entry to the `AIEditorWorkspace`.
- Logic for task list: Find the first/latest log entry for each keyword to determine if it's a creation or optimization task.

### 2. Update `src/app/creation/ai-editor-workspace.tsx`
- Add a `mode` or `actionType` prop.
- Implement conditional rendering:
    - If `mode === 'Erstellung'`: Render a clean markdown preview or text area for the new content.
    - If `mode === 'Optimierung'`: Continue using `ReactDiffViewer`.

### 3. Logic for "Latest Action"
- In `CreationPage`, we need to determine the type for each keyword in the list.
- We can check the `ContentLog` entries. If the latest entry for a keyword has `Action_Type === 'Erstellung'`, it's a creation task.
